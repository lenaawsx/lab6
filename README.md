Лабораторная работа №6. Асинхронный JavaScript: API и хранилище
1. Что делает fetch?

fetch() — это встроенный метод JavaScript, который позволяет отправлять HTTP-запросы к серверу и получать данные без перезагрузки страницы.

Он работает асинхронно и возвращает объект Promise.

Пример:

fetch("https://jsonplaceholder.typicode.com/posts")
  .then(r => r.json())
  .then(data => console.log(data));

2. Зачем нужны async/await?

async/await — это современный синтаксис для удобной работы с асинхронным кодом.

Зачем они нужны:

делают код более понятным, похожим на обычный последовательный;

убирают «пирамиды ада» из .then();

позволяют писать запросы «шаг за шагом» и ловить ошибки через try/catch.

Пример:

async function load() {
  try {
    let r = await fetch(url);
    let j = await r.json();
    console.log(j);
  } catch(e) {
    console.log("Ошибка загрузки");
  }
}

3. Как работает localStorage?

localStorage — это встроенное хранилище браузера, куда можно сохранять данные между перезагрузками страницы.

Особенности:

данные хранятся, пока пользователь сам не удалит (или кодом);

всё сохраняется в виде строк;

ограничение — примерно 5–10 МБ.

Пример:

localStorage.setItem("theme", "dark");
let t = localStorage.getItem("theme"); // "dark"
